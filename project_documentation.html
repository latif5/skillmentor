Version: 1.0.0 | Date: 2026-01-12

Status: MVP Completed & Premium Redesign Implemented

1. Executive Summary

SkillMentor is a modern, AI-enhanced learning management system (LMS) designed to empower students and lifelong learners. It combines the utility of traditional flashcards with advanced study modes, collaborative class features, and a premium "glassmorphism" aesthetic.

The platform is built for speed, visual appeal, and ease of use, leveraging a real-time backend (Convex) and a robust React-based frontend (Next.js).

2. Product Requirements Document (PRD)

2.1 Problem Statement

Traditional study tools are often clunky, visually unappealing, or lack collaborative features. Students need a platform that makes studying engaging through gamification, aesthetic design, and seamless sharing capabilities.

2.2 Goals

  • Engagement: Increase study time through visually pleasing UI and gamified modes (Match, Test).
  • Collaboration: Enable students to share knowledge easily via Classes.
  • Simplicity: Provide an intuitive "zero-friction" onboarding experience.
  • Performance: Ensure real-time updates and fast transitions using modern tech.

2.3 Target Audience

Segment Description
Students (High School/University) Primary users. Need flashcards, practice tests, and mobile accessibility for exams.
Lifelong Learners Self-paced learners focusing on languages or professional certifications.
Teachers/Educators Create curriculum, manage classes, and distribute study sets to groups.

3. Actors & Roles

3.1 System Actors

  • Guest: A visitor who has not logged in. Can view the Landing Page, Features, and Pricing. Restricted from viewing specific study sets or classes.
  • Authenticated User: A logged-in user with a comprehensive dashboard.
  • System (SkillMentor Platform): Handles authentication, data persistence, payment processing (mock), and AI content generation (planned).

3.2 Functional Roles

Roles define permissions within the application context.

Role Permissions
Student (Default)
  • Create/Edit/Delete own Study Sets
  • Create/Edit/Delete own Folders
  • Join Classes via invite
  • Study all public sets
Teacher
  • All Student permissions
  • Create and Manage Classes
  • View Class Analytics (Future)
  • Manage Student Rosters
Admin System oversight, user management, and global content moderation.

4. Feature Specifications

4.1 Study Sets Core

The fundamental unit of learning on SkillMentor.

  • Creation: Rich text editor for Term and Definition pairs. Supports visibility settings (Public/Private).
  • Flashcards Mode: Classic flip-card interface with keyboard navigation.
  • Learn Mode: Adaptive questioning to reinforce weak terms.
  • Test Mode: Generated quizzes (Multiple choice, True/False, Written) with instant grading.
  • Match Mode: Gamified timed challenge to match terms to definitions.

4.2 Classes & Collaboration Core

Enables social learning and content distribution.

  • Class Creation: Users (Teachers) can create named classes with descriptions.
  • Membership: Users can join classes.
  • Content Sharing: Study sets can be linked to specific classes for all members to access.

4.3 Organization Core

Tools to keep high-volume learners organized.

  • Folders: Group related study sets together (e.g., "Biology 101", "Spanish Vocab").
  • Dashboard: Centralized hub showing "Your Study Sets", "Your Classes", and quick actions.

4.4 Authentication & Security Integration

  • Provider: Clerk (Third-party Auth).
  • Methods: Email/Password, Social Login (Google, GitHub).
  • Protection: Middleware protects `/dashboard`, `/sets`, and `/classes` routes.

5. Technical Architecture

5.1 Technology Stack

Layer Technology Justification
Frontend Framework Next.js 14 (App Router) Server-side rendering, robust routing, and industry standard for React apps.
Styling Tailwind CSS Utility-first CSS for rapid UI development and custom design systems.
Backend / DB Convex Real-time database, backend-as-a-service, automatic caching, and type-safe API.
Authentication Clerk Seamless, secure, and pre-built UI components for auth flows.
Icons Lucide React Clean, modern SVG icons that match the premium aesthetic.

6. Data Model (Convex Schema)

The database is relational, enforced via application logic and Convex schemas.

Users

  • `tokenIdentifier` (String): Link to Clerk ID
  • `name` (String)
  • `email` (String)
  • `role` (String): e.g., 'teacher', 'student'

StudySets

  • `ownerId` (User ID)
  • `title` (String)
  • `description` (String)
  • `visibility` (String): 'public' or 'private'

Terms

  • `studySetId` (StudySet ID)
  • `term` (String)
  • `definition` (String)

Classes

  • `ownerId` (User ID)
  • `name` (String)
  • `description` (String)

7. UI/UX Design System

The application follows a "Premium Modern" aesthetic.

  • Color Palette: Dominant Indigo/Violet gradients (`from-indigo-600 to-purple-600`) against clean White/Zinc backgrounds.
  • Typography: Inter font family. Tight tracking on headings for a modern look.
  • Glassmorphism: Use of `backdrop-blur`, semi-transparent white backgrounds, and subtle borders.
  • Motion: Staggered entrance animations (`animate-fade-up`), hover lift effects, and smooth transitions.
